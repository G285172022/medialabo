本テーマの学習目的

  • HTML によって Web ページの構造を記述する基本的な方法がわかる。
  • CSS によって Web ページの見た目を設定する基本的な方法がわかる。
  • JavaScript によって Web ページのインタラクティブな処理を記述する基本的な方法がわかる。
  • HTML, CSS, JavaScript の構文リファレンスから，構文の書式や使用方法の情報を入手できる。
  • HTML, CSS, JavaScript, 画像・音声・動画ファイルを組み合わせて簡単なWebページが作成できる。

プロジェクト課題について

プロジェクト課題では、NHKの番組表の検索を実施する。 表示する内容は、検索欄、検索結果の件数、検索結果の番組情報の一覧である。
検索方法は、チェックボックスでチャンネルを、プルダウンメニューで番組のジャンルを指定するようにした。 検索するチャンネルが一つも選択されないまま検索ボタンが押されたときは、選択するように促すメッセージを表示する。 また、2つのチャンネルが選択されたときも、正しくすべての検索結果が表示される。
検索結果は、table要素を用いて番組一つに一つの表で番組情報の一覧として表示している。取得した番組情報に空欄の要素があればその要素は表には表示されないようにした。 また、取得した番組情報のlengthから検索結果の件数を表示している。
検索の条件に当てはまる番組情報が一つもない場合も、エラーは出ずに「検索結果は0件」とだけ表示されるようにした。

プロジェクト課題で作成したページ: project.html

数当てゲーム

この課題では、ランダムに生成される答えの数字を、入力した数字との大小比較の情報をもとに当てるゲームの作成を実施する。
第3回ではJavaScriptを用いて数字の生成と入力した数字と生成した数字の大小の判定処理を実装した。ここでは判定の結果をWebページ上に表示することができないため、結果はコンソールに表示した。
第4回ではDOMプログラミングでHTMLを編集することで、予想回数や予想の数字、第3回で実装した判定処理の結果をWebページ上に表示できるようにした。
第5回では数字の入力欄と回答ボタンを設置して、イベント処理を用いて入力欄に入力された数字と答えの数字との大小比較の処理をボタンで呼び出すという処理を実装した。

数当てゲームのページ: kazuate.html

工夫点、苦心点、強調点、感想

工夫した点及び苦労した点としては、通信で受信したデータの処理である。
過去にもHTMLやCSSを書く機会はあったが、HTMLやCSSの基本的な要素に軽く触るだけのサイトを作っただけであったので、JavaScriptによるWebプログラミングを含んだ本格的なWebサイト制作は今回が初めてだった。今回始めて使ったJavaScriptのプログラミングは、基本的には他のプログラミング言語と同じような書き方をすることから、今まで学んできたプログラミングの知識を活かすことができ、スムーズに書くことができた。
しかし、axiosのような今回初めて習う要素を使いこなすのには、多くの試行錯誤を必要とした。プロジェクト課題において、通信で受信したデータの処理で改善しなければならなかった点は、3点ある。
1つ目は、2つのaxiosのリクエストを送る場合があり、その際にサーバーから送られてくるオブジェクトのデータを、オブジェクト内の配列の名前によって事前に定義した正しいundefinedオブジェクトの変数に格納できるようにしたことである。
2つ目は、表示処理の検索結果の件数を表示する部分でオブジェクト内の配列のlengthを用いていることから、検索の条件に当てはまるデータが一つもない場合にそのまま表示処理を実行するとオブジェクトの変数がundefinedであることでエラーとなってしまうため、データがある場合にのみ件数表示処理で配列を参照するようにしたということである。
3つ目は、2つのリクエストを送信した際に、受信したタイミングで表示処理を呼び出すと、先に受信した方のデータのみ表示されてしまったり、表示処理が2回呼び出されて検索結果が2重に呼び出されてしまったりしたため、2回目に受信したときだけ表示処理を呼び出すようにしたことである。
以上3点を試行錯誤して改善したということが、工夫した点及び苦労した点である。
この実験の感想として、6回の講義でWebサイトを作っている要素を学び、その要素を組み合わせた一つのWebサイトを作るというのがとてもいい経験となったと感じた。特に、プロジェクト課題においてのデータを受信したあとの処理を試行錯誤しながら構築したことは、今後非同期処理をするプログラムを書く際に設計を行う上での参考となる経験となったのではないかと考える。
また、検索結果が思い通りに表示され、プロジェクト課題のページが完成したときは、大きな達成感を得ることができた。
今後、この実験で得られたWebページを構成している要素についての知識やWebプログラミングに関する知識を今後の研究や実務、生活などに活かしていきたいと思う。
